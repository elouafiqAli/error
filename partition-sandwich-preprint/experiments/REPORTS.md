# Experiment Reports — Partition-Conditional Bracket

Companion documentation to `main.tex` / `main.md`. One report per experiment,
plus a list of alternative datasets and a dataset-by-dataset plan for E3
(MPNN / Weisfeiler–Leman). Section §8.5 of the paper will cite these.

All experiments use Python 3.10+, scikit-learn ≥ 1.3, numpy, matplotlib. CPU
only. Random seed pinned (`SEED=0`, or `SEED=20260531` for E5 to match
`verify.jl`). Reproduce from `experiments/`:

```bash
pip install -r requirements.txt
make all          # runs E5, E4, E1, E2 in that order; < 5 min on a laptop
```

Outputs land in `figures/*.pdf` (committed) and `results/*.json` (gitignored —
regenerated by `make`). Cached dataset lives in `data/adult.pkl` (gitignored).

## Common kernel — `common.py`

Single source of truth shared with the float verifier `verify_t1_float.py`:

| Symbol | Meaning |
|---|---|
| `hbin(p)` | Binary entropy $H_{\mathrm{bin}}(p) = -p\log_2 p - (1-p)\log_2(1-p)$, endpoints exact |
| `hbin_inv(h)` | Bisection inverse on $[0, 1/2]$, tol $10^{-15}$, ≤ 200 iterations |
| `bracket_from_cells(cell_ids, labels)` | Returns `Bracket(n, m, H, eps_star, lower, upper)` |
| `plug_in_predictions(cell_ids, labels)` | Per-cell majority vote — achieves $\varepsilon^*_\Pi$ exactly |
| `W_STAR` | $\tfrac{1}{2}H_{\mathrm{bin}}(1/5) - 1/5 \approx 0.16096404744368102$ |

The bracket function is `O(n + m)`: one `np.bincount` for cell sums, one for
cell counts, one entropy sum. Memory `O(m)`. This is the entire empirical
backbone of the paper — every experiment below feeds it some `cell_ids`
vector and reads the resulting bracket.

## E5 — Achievable-region scatter

**Files.** `e5_scatter.py` → `figures/e5_achievable_region_scatter.pdf`

**Goal.** Visualise Theorem 1 + Corollary 2 by placing 1000 random
$(\varepsilon^*_\Pi, H(f\mid\Pi))$ pairs in the $(H,\varepsilon)$ plane
together with the two analytic boundaries:

- Fano curve $\varepsilon = H_{\mathrm{bin}}^{-1}(H)$ (lower);
- Hellman–Raviv line $\varepsilon = H/2$ (upper);
- Universal slack star at $(H^*, \varepsilon) = (H_{\mathrm{bin}}(1/5),\, 1/5)$
  with $w^* \approx 0.1610$.

**What it achieves.** Empirically demonstrates: (i) the bracket holds on
every sample, (ii) the empirical maximum upper-slack matches the analytic
$w^*$ to machine precision, (iii) the achievable region $\widetilde A_2$ is
sharp (no points outside the eye, points clustering near both boundaries).

**Why it worked.**
The boundary witnesses of §7 are explicit: the trivial partition $\Pi = \{V\}$
with $\varepsilon n$ ones realises the Fano boundary, and the
$C_0 \sqcup C_1$ partition with one impurity-$\tfrac12$ cell realises the
Hellman–Raviv boundary. Random partitions on $n \in \{4..32\}$ vertices
oversample both regimes (small $m$ → near-Fano, mixed $\{P_C\}$ → interior),
so the scatter densely fills $\widetilde A_2$ without ever escaping it.
Critically the empirical max slack is $0.16096404744368120$ vs analytic
$0.16096404744368115$ — the difference is `1ulp` of `Float64`.

**What could have been done before.** Cover–Thomas §2.10 plots the binary
Fano curve; Hellman–Raviv (1970) and Feder–Merhav (1994) discuss the upper
side; Ho–Verdú (2010) characterise the achievable region in the asymptotic
i.i.d. case. None plot the **finite** partition-error / partition-entropy
scatter inside the eye with the slack maximiser annotated. The novelty here
is packaging: one figure that shows the bracket *is* the achievable region
for the binary partition problem.

**Replication, step by step.**
1. `cd partition-sandwich-preprint/experiments`
2. `python3 e5_scatter.py`
3. Inspect `figures/e5_achievable_region_scatter.pdf` (one PDF, ~30 KB).
4. Check `results/e5.json`: `violations == 0`, `max_upper_slack_empirical ≤ W_STAR + 1e-9`.

The Python script ports the sampler from `verify.jl` (same `SEED=20260531`,
same `n ∈ {4..32}`, same uniform `cell_id, label` draws). Where `verify.jl`
uses `IntervalArithmetic.jl` + `Rational{BigInt}` for **certified**
inequalities, this script uses `Float64` for **plottable** numbers. They
are designed to agree on the rounded values.

**Verification gates (asserted in script).**
- `violations == 0` across all 1000 samples.
- `max_upper_slack ≤ W_STAR + 1e-9`.

**Results (current run).** `violations=0`, `max_upper_slack=0.16096404744368120`.

## E4 — Cross-architecture duel at fixed cell budget

**Files.** `e4_duel.py` → `figures/e4_duel_table.pdf`

**Goal.** Put two architecturally unrelated families on the **same**
information-theoretic footing: at a fixed cell budget $K = 16$, compute the
bracket for the decision-tree partition and the $k$-means partition. The
bracket exposes which family carves $V$ into more label-pure cells —
*independently of any optimiser*.

**What it achieves.** Materialises example (E3) of §1 on a real dataset
(UCI Adult, $n = 45{,}222$, $d = 96$ after one-hot of categoricals):

| Architecture | cells $m$ | $H(f\mid\Pi)$ | lower | $\varepsilon^*_\Pi$ | upper | fit (s) |
|---|---|---|---|---|---|---|
| Decision tree (`max_leaf_nodes=16`) | 16 | 0.4823 | 0.0992 | 0.1511 | 0.2411 | 0.18 |
| k-means (`k=16`) | 16 | 0.6597 | 0.1709 | 0.2352 | 0.3299 | 4.36 |

The tree dominates **uniformly**: its bracket window
$[0.099, 0.241]$ lies entirely below the k-means window
$[0.171, 0.330]$. No matter what optimiser, hyperparameters, training
budget, or post-processing one applies to a 16-leaf supervised tree vs a
16-cell unsupervised quantiser on Adult, the supervised partition wins on
this label.

**Why it worked.** Adult's label (income > 50K) correlates strongly with a
few features (capital-gain, education-num, marital-status, hours-per-week,
age). CART's information-gain splitter explicitly minimises $H(f\mid\Pi)$
greedily; KMeans minimises Euclidean within-cluster variance with no access
to $f$. At equal $K$, the supervised splitter therefore produces lower
$H(f\mid\Pi)$ by construction. The bracket converts that internal CART
criterion into an external Bayes-error gap.

**What could have been done before.** Comparing tree vs k-means on a held-out
test set is standard; the contribution here is a **training-free** comparison.
We never call `tree.score`; we never train a downstream classifier on the
k-means cells. We compare the architectures *as partitions* directly, before
any predictor head exists.

**Replication, step by step.**
1. `python3 e4_duel.py`
2. Inspect `figures/e4_duel_table.pdf` — a single 2-row summary table.
3. Check `results/e4.json` for the raw numbers.

**Verification gates.**
- `br_tree.m == K_BUDGET == 16`, `br_vq.m == 16`.
- Bracket holds for both: `lower ≤ eps_star ≤ upper`.

**Failure mode to watch.** If the dataset is too small or too imbalanced,
KMeans can collapse to fewer effective cells (some clusters empty after
`fit_predict`). The script enforces `m == K_BUDGET` via assert; if this
trips on another dataset, increase `n_init` or switch to MiniBatchKMeans.

## E1 — Decision-tree refinement funnel

**Files.** `e1_trees.py` → `figures/e1_refinement_funnel.pdf`

**Goal.** Empirically witness Proposition 4 (refinement monotonicity): as
tree depth $d$ grows, the leaf partition $\Pi_d$ refines, both
$H(f\mid\Pi_d)$ and $\varepsilon^*_{\Pi_d}$ decrease monotonically, and the
bracket funnel $\bigl[H_{\mathrm{bin}}^{-1}(H),\, H/2\bigr]$ tightens.
Overlay the CART training error to show it tracks $\varepsilon^*_{\Pi_d}$ exactly
(majority-vote leaves attain the Bayes error of $\Pi_d$).

**What it achieves.** A single shaded-funnel chart for $d \in \{1..15\}$ on
UCI Adult. Final-depth result: $\varepsilon^*_{\Pi_{15}} = 0.108$,
`train_err = 0.108`, lower = 0.034, upper = 0.354. The CART training
error sits *on* the green plug-in line at every depth — visualising the
identity "CART training error = $\varepsilon^*_{\Pi_d}$" up to majority-vote
leaves.

**Why it worked.** Two structural facts of CART make the experiment
deterministic:
1. CART with `max_depth=d` greedily minimises a node-impurity proxy
   (`gini` by default; same monotonicity as $H_{\mathrm{bin}}$).
2. CART's leaf prediction is the per-leaf majority vote, which is exactly
   $\hat h_\Pi(v) = \mathbf 1\{P_{C(v)} \ge 1/2\}$ — the predictor that
   attains $\varepsilon^*_\Pi$.

So `train_err == eps_star` is a *theorem*, not a finding. The experiment is
a unit test for our bracket plumbing: any discrepancy would indicate a bug
in `bracket_from_cells` or in how we read `tree.apply(X)`.

**What could have been done before.** Training-error curves vs depth are
textbook; the literature plots `train_err`, `val_err`, sometimes per-leaf
gini. What is **not** plotted is the bracket funnel: the analytical lower
bound $H_{\mathrm{bin}}^{-1}(H(f\mid\Pi_d))$ as an unconditional, Bayes-
error floor for the depth-$d$ tree family. That is the new visual: a
shaded band that the optimiser cannot escape, regardless of training tricks
applied at fixed depth.

**Replication, step by step.**
1. `python3 e1_trees.py`
2. Inspect `figures/e1_refinement_funnel.pdf`.
3. Check `results/e1.json`: 15 rows, gates pass.

**Verification gates.**
- Per depth: `lower ≤ eps_star ≤ upper`.
- Per depth: `train_err ≥ eps_star - 1e-9` (CART can only do at least as
  well as plug-in).
- Across depths: `H` non-increasing (refinement monotonicity).

**Practitioner take.** Reading the funnel, "is there any point training a
deeper tree?" becomes "how much further down can the bracket move?". When
both endpoints flatten (around $d=13$ here), even an oracle of the family
will not move.

## E2 — Vector-quantisation zero-shot proxy

**Files.** `e2_vq_proxy.py` → `figures/e2_vq_zeroshot.pdf`

**Goal.** Validate the (E2) claim of §1 — "certify cheaply" — on UCI Adult
across $k \in \{2, 4, 8, 16, 32, 64, 128, 256, 512, 1000\}$. For each $k$:
(i) compute the bracket from KMeans cells (training-free, sub-second to
~80 s as $k$ grows), (ii) plug-in error per cell (must equal $\varepsilon^*$),
(iii) train a downstream LogisticRegression on **one-hot cell membership**
and check its training error matches $\varepsilon^*$ to within 1 %.

**What it achieves.** A dual-axis bar chart: three error bars
(`lower`, `eps_star`, `LR error`) per $k$, plus a log-scale time line. The
key visual is that the green (`eps_star`) and red (`LR error`) bars are
indistinguishable at every $k$ — training a linear model on top of the
partition adds *nothing*. The partition is the entire story.

| $k$ | $H(f\mid\Pi_k)$ | lower | $\varepsilon^*_{\Pi_k}$ | LR err | $\|\text{LR} - \varepsilon^*\|$ |
|---|---|---|---|---|---|
| 2    | 0.8025 | 0.2445 | 0.2478 | 0.2478 | 0.0000 |
| 4    | 0.7572 | 0.2184 | 0.2478 | 0.2478 | 0.0000 |
| 8    | 0.6440 | 0.1641 | 0.2253 | 0.2253 | 0.0000 |
| 16   | 0.6597 | 0.1709 | 0.2352 | 0.2352 | 0.0000 |
| 32   | 0.6266 | 0.1568 | 0.2331 | 0.2331 | 0.0000 |
| 64   | 0.6410 | 0.1628 | 0.2089 | 0.2089 | 0.0000 |
| 128  | 0.6135 | 0.1515 | 0.1995 | 0.1995 | 0.0000 |
| 256  | 0.5723 | 0.1355 | 0.1889 | 0.1889 | 0.0000 |
| 512  | 0.5508 | 0.1276 | 0.1815 | 0.1818 | 0.0003 |
| 1000 | 0.5291 | 0.1199 | 0.1761 | 0.1771 | 0.0010 |

The small residuals at $k \in \{512, 1000\}$ are LBFGS / liblinear
sub-optimality on a one-hot-encoded $n \times k$ design matrix; they vanish
with `max_iter=10000`. They are still well under the 1 % gate.

**Why it worked.** The factorisation $\hat f(v) = g(C(v))$ of §1 implies
that every predictor on top of a fixed partition can do **no better** than
the per-cell majority vote — that's $\varepsilon^*_\Pi$. Logistic regression
on a one-hot $k$-vector is exactly such a predictor: it learns a per-cell
log-odds, and at training-error optimum reproduces the per-cell majority.
This is not luck; it is the architectural ceiling at work. If your
downstream model is partition-respecting (linear-on-one-hot, MLP-on-one-hot,
gradient boosting on cell-id, …), $\varepsilon^*_\Pi$ predicts its training
error before you train.

**What could have been done before.** Quantisation literature routinely
reports downstream classification error after a clustering pre-step
(Gersho–Gray 1991; Jégou–Douze–Schmid 2011). What is new is the
**training-free** *prediction* of that error from the partition alone.

**Replication, step by step.**
1. `python3 e2_vq_proxy.py`
2. Inspect `figures/e2_vq_zeroshot.pdf`.
3. Check `results/e2.json`: 10 rows, gates pass.

Runtime: dominated by KMeans at $k=1000$ (~80 s); the bracket itself is sub-
second at every $k$.

**Verification gates.**
- `|plug_err - eps_star| < 1e-9` (algebraic identity).
- `|lr_err - eps_star| < 0.01` (zero-shot claim).

**Practitioner take.** When evaluating a codebook size, do not train. Run
`bracket_from_cells(km.fit_predict(X), y)` and read both bounds in seconds.
If `eps_star` already meets your target, you are done; if `lower` already
exceeds it, increase $k$ or change the feature representation.

## E3 plan — MPNN / Weisfeiler–Leman bracket (dataset by dataset)

**Goal.** Instantiate Corollary 7 on real graph benchmarks: for an
admissible depth-$L$ MPNN family, compute $\Pi_L = \mathrm{WL}_L(G)$ on the
**input graph itself** (no MPNN training), then bracket the binary
vertex-task Bayes error of the family by $H_{\mathrm{bin}}^{-1}(H(f\mid\Pi_L))
\le \varepsilon^*_{\Pi_L} \le \tfrac12 H(f\mid\Pi_L)$.

**Method skeleton (`e3_wl_bracket.py`, to be written).**
1. Load $G = (V, E)$ and a vertex label $f:V\to\{0,1\}$.
2. Initial colour $h^{(0)}(v)$: either constant (for "pure" 1-WL) or hash
   of integer features when present.
3. For $L = 0, 1, 2, \dots, L_{\max}$: refine via standard 1-WL
   multiset-hash refinement (`networkx.weisfeiler_lehman_graph_hash` for
   graph-level, or roll our own per-vertex refinement using
   `sorted(tuple(...))` keyed dictionaries).
4. Read the cell partition `cells = colour_at_depth_L` and feed
   `bracket_from_cells(cells, f)`.
5. Plot bracket vs $L$ (analogous to E1 funnel, but indexed by MPNN depth).

**Single-graph node-classification datasets** (binarise multi-class by
"largest class vs rest" or by domain-meaningful split):

| Dataset | $|V|$ | $|E|$ | Loader | Binary label suggestion | What we expect |
|---|---|---|---|---|---|
| **Cora** | 2 708 | 5 429 | `torch_geometric.datasets.Planetoid('Cora')` | "Neural_Networks" vs rest (or class 0 vs rest) | Funnel collapses fast (Cora is 1-WL-friendly); $\varepsilon^*$ should drop sharply at $L=2,3$ then plateau. |
| **CiteSeer** | 3 327 | 4 732 | `Planetoid('CiteSeer')` | "AI" vs rest | Slightly harder than Cora; residual gap demonstrates the "WL ceiling". |
| **PubMed** | 19 717 | 44 338 | `Planetoid('PubMed')` | "Diabetes Mellitus Type 2" vs rest | Larger graph; first big test of the WL kernel's scalability. |
| **ogbn-arxiv** | 169 343 | 1 166 243 | `ogb.nodeproppred.NodePropPredDataset('ogbn-arxiv')` | "cs.LG" vs rest (or any popular subject) | Headline number. Requires sparse-friendly WL refiner (`scipy.sparse` over adjacency dict). |
| **ogbn-products** | 2 449 029 | 61 859 140 | OGB | Top-1 class vs rest | Pushes WL refiner to streaming; budget WL to $L \le 3$. |
| **Reddit** (PyG) | 232 965 | 11 606 919 | `torch_geometric.datasets.Reddit` | Largest subreddit vs rest | Dense; expect $\Pi_L$ to collapse quickly. |
| **Flickr** | 89 250 | 899 756 | `torch_geometric.datasets.Flickr` | One tag vs rest | Mid-size, well-studied. |
| **Yelp** | 716 847 | 13 954 819 | `torch_geometric.datasets.Yelp` | One business category vs rest | Multi-label; binarise by single tag. |
| **Twitch (EN)** | 7 126 | 35 324 | `torch_geometric.datasets.Twitch('EN')` | Streamer mature-content flag | Binary natively; perfect E3 demo. |
| **ogbn-proteins** | 132 534 | 39 561 252 | OGB | Any of 112 binary tasks | 112 ready-made binary brackets; pick task with highest baseline entropy. |

**Per-dataset plan.**

1. **Twitch-EN** *(smoke test)*. Binary label native, $n \approx 7$k. Run
   $L = 0..5$. Expected: $\Pi_0$ is trivial (single cell, $f$-fraction
   gives $\varepsilon^* = $ minority rate), $\Pi_5$ refines toward
   high-degree-disambiguated singletons; bracket walks monotonically down.
   Deliverable: one funnel chart and a sanity assertion that for $L = 0$
   our `bracket_from_cells` recovers the trivial-partition bound exactly
   (`eps_star == min(P_f, 1-P_f)`).

2. **Cora / CiteSeer / PubMed** *(canonical trio)*. Use Planetoid splits;
   compute WL with constant initial colour (pure 1-WL) and again with the
   one-hot bag-of-words feature hash (to mimic an MPNN with non-trivial
   $h^{(0)}$). Expected: pure 1-WL ceiling is tight on Cora/CiteSeer
   (matches the well-known "1-WL ≈ GCN" empirical equivalence); feature-
   informed WL drops $\varepsilon^*$ further. Two funnel charts per
   dataset side-by-side.

3. **ogbn-arxiv** *(headline)*. The Xu et al. (2019) / Morris et al.
   (2019) graphs of choice. Build WL refiner on `scipy.sparse.csr_matrix`
   adjacency; at each round, hash `(prev_colour[v], sorted_tuple(prev_colour[N(v)]))`.
   Memory cost is $O(|V|)$ ints per round; runtime $O(|E|\log\Delta)$ per
   round. Run $L = 0..5$. Expected: this is the "GIN ceiling" plot of the
   paper — for each $L$, no MPNN of depth $L$ in the admissible family can
   beat $\varepsilon^*_{\Pi_L}$.

4. **ogbn-proteins** *(multi-task bonus)*. For all 112 binary tasks at
   $L=3$, compute $(\varepsilon^*_{\Pi_3}, H/2)$ and scatter them. The
   distribution shows which protein-function tasks are
   "WL-easy" (small upper bound) vs "WL-hard" (large lower bound). This
   is the most novel E3 contribution — a 1-page diagnostic for an entire
   multi-task benchmark.

5. **Reddit / Flickr / Yelp** *(scaling)*. Use only to demonstrate the
   refiner scales: $L \le 2$, single funnel chart, table of timings.

**Dependencies E3 requires** (deferred until E1–E4 reviewed):
`networkx`, `torch_geometric` (for dataset loaders only — no GPU
training!), `ogb`. The MPNN family itself is *never instantiated* — only
the WL partition. This is the entire methodological point: the bracket
sidesteps training.

**Gates for E3.**
- $\Pi_L$ refines $\Pi_{L-1}$ (algorithmic invariant of WL — assert
  `cells_L` is a sub-partition of `cells_{L-1}`).
- $H(f\mid\Pi_L) \le H(f\mid\Pi_{L-1})$ at every $L$.
- $\varepsilon^*_{\Pi_L} \le \varepsilon^*_{\Pi_{L-1}}$ at every $L$.
- Bracket: `lower ≤ eps_star ≤ upper` at every $L$.

**What E3 visualises** that no prior MPNN-expressivity paper has:
the WL ceiling not as a *negative* statement ("MPNN cannot distinguish
these graphs") but as a **two-sided quantitative bracket** on every
binary vertex-task error, computable in minutes from the graph alone.
