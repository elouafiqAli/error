## E2b — Marginal-aware refinement on real data

### Targets the reviewer concern
*"$w^* \approx 0.161$ is a worst-case slack; is it actually narrower
on real data?"* — answered by **Proposition 6** (paper §4) but the
paper currently shows only a synthetic table. E2b demonstrates the
real-data reduction.

### Files to create
- `experiments/e2b_marginal_aware.py`
- `experiments/figures/e2b_marginal_slack.pdf`
- `experiments/results/e2b.json` (gitignored)

### Method

1. Reuse `datasets.py::load_dataset(name)` for `"adult"`, `"spambase"`,
   `"phishing"`. Add the latter two loaders if not present
   (`fetch_openml('spambase')`, `fetch_openml('PhishingWebsites')`).
2. For each dataset:
   1. Compute the global marginal $\pi = \bar{f}$ and
      $\pi_* = \min(\pi, 1-\pi)$.
   2. Build a CART partition $\Pi_T$ at `max_leaf_nodes = 16` (same
      knob as E4).
   3. Build a $k$-means partition $\Pi_Q$ at `k = 16`.
   4. For each partition compute the universal-slack bound $w^*$
      and the marginal-aware slack $w^*(\pi_*)$ from the
      piecewise closed form:
      ```python
      threshold = 0.5 * hbin(1/5)               # ≈ 0.3610
      if pi_star >= threshold:
          w_marg = W_STAR
      else:
          w_marg = pi_star - hbin_inv(2 * pi_star)
      ```
   5. Also compute the realised upper-side slack
      $\tfrac{1}{2}H(f\mid\Pi) - \varepsilon^*_\Pi$ on the actual
      partition; report whether it falls below $w^*(\pi_*)$.

### Output schema (`results/e2b.json`)

```json
{
  "datasets": [
    {
      "name": "adult", "n": 45222, "pi": 0.2393, "pi_star": 0.2393,
      "tree": {"H": 0.482, "slack_realised": 0.123, "w_star": 0.161, "w_marg": 0.131},
      "vq":   {"H": 0.660, "slack_realised": 0.094, "w_star": 0.161, "w_marg": 0.131}
    },
    {"name": "spambase", "...": "..."},
    {"name": "phishing", "...": "..."}
  ],
  "gates": {"all_slack_le_w_star": true, "all_slack_le_w_marg": true}
}
```

### Figure
Single bar chart with two grouped bars per dataset:
**universal slack** (grey) vs **marginal-aware slack** (blue), with
the realised slack as a horizontal tick. Title: *"Marginal-aware
slack shrinks the bracket on every real binary label we tested."*

### Verification gates
- `slack_realised <= W_STAR + 1e-9` for every (dataset, architecture).
- `w_marg <= W_STAR + 1e-9` always; **strict** inequality on at least
  one dataset with $\pi_* < 0.361$.
- The script asserts `slack_realised <= w_marg + 1e-9` (the
  marginal-aware bound is also realised).

### Runtime budget
< 20 s on a laptop (3 datasets × 2 architectures × tiny fits).

## E3 — MPNN / WL bracket on real graphs

### Targets the reviewer concern
*"You claim the bracket lifts to GNNs but only show the
$C_4$-alternating one-line example."* — answered by Corollary 7
+ Proposition 4 but with no real-graph evidence yet. E3 produces
the headline empirical plot for the GNN pivot.

### Files to create
- `experiments/e3_wl_bracket.py`
- `experiments/figures/e3_<dataset>_funnel.pdf` (one per dataset)
- `experiments/figures/e3_proteins_scatter.pdf` (ogbn-proteins
  bonus)
- `experiments/results/e3.json`

### Dependencies (deferred until E1–E4 reviewed)
```
networkx>=3.0
torch-geometric>=2.4   # dataset loaders only — NO GPU training
ogb>=1.3
```
The MPNN family itself is **never instantiated** — only the WL
partition.

### Method (per dataset)

1. Load $G = (V, E)$ and a vertex label $f : V \to \{0,1\}$.
2. Initial colour $h^{(0)}(v)$:
   - **pure 1-WL**: constant on all vertices;
   - **feature-informed**: hash of integer features when present.
3. For $L = 0, 1, \dots, L_{\max}$, refine WL via the standard
   multiset-hash:
   ```python
   def refine(colours, adj):
       new = {}
       for v in V:
           key = (colours[v], tuple(sorted(colours[u] for u in adj[v])))
           new[v] = hash(key)
       # canonicalise: relabel to consecutive ints
       remap = {h: i for i, h in enumerate(sorted(set(new.values())))}
       return [remap[new[v]] for v in V]
   ```
   For sparse graphs (`ogbn-arxiv`+), back the adjacency by
   `scipy.sparse.csr_matrix` and iterate `indptr`/`indices` slices.
4. Feed the cell vector at depth $L$ into `bracket_from_cells`.
5. Plot funnel: $L$ on x-axis, $[\mathrm{lower},
   \varepsilon^*_{\Pi_L}, \mathrm{upper}]$ on y-axis, shaded band
   between bracket endpoints, $\varepsilon^*$ as a solid line.

### Datasets (in execution order)

| Order | Dataset | $|V|$ | $L_{\max}$ | Binary label | Loader |
|---|---|---|---|---|---|
| 1 (smoke) | **Twitch-EN** | 7 126 | 5 | mature-content flag (native) | `torch_geometric.datasets.Twitch('EN')` |
| 2 | **Cora** | 2 708 | 5 | "Neural_Networks" vs rest | `Planetoid('Cora')` |
| 3 | **CiteSeer** | 3 327 | 5 | "AI" vs rest | `Planetoid('CiteSeer')` |
| 4 | **PubMed** | 19 717 | 4 | "Diabetes Mellitus Type 2" vs rest | `Planetoid('PubMed')` |
| 5 (headline) | **ogbn-arxiv** | 169 343 | 4 | "cs.LG" vs rest | OGB |
| 6 (bonus) | **ogbn-proteins** | 132 534 | 3 | all 112 binary tasks | OGB |

Stop at step 5 if the funnel for ogbn-arxiv is the desired
headline; step 6 is the most novel deliverable (multi-task
scatter) and is worth running last.

### Output schema (`results/e3.json`)

```json
{
  "datasets": [{
    "name": "twitch_en", "n_V": 7126, "n_E": 35324, "pi": 0.55,
    "depths": [
      {"L": 0, "m": 1,    "H": 0.99, "lower": 0.49, "eps_star": 0.45, "upper": 0.495},
      {"L": 1, "m": 1240, "H": 0.71, "lower": 0.20, "eps_star": 0.31, "upper": 0.355},
      ...
    ],
    "monotone_H": true, "monotone_eps": true, "bracket_holds": true
  }, ...],
  "ogbn_proteins_scatter": [
    {"task": 0, "L": 3, "lower": 0.07, "eps_star": 0.11, "upper": 0.27}, ...
  ]
}
```

### Figures
- One funnel per dataset (5 PDFs, ≤ 30 KB each).
- One scatter for `ogbn-proteins`: x-axis = $H/2$ (upper bound),
  y-axis = $\mathrm{Hinv}(H)$ (lower bound), one point per task,
  coloured by realised $\varepsilon^*$.

### Verification gates
- `cells_L` is a sub-partition of `cells_{L-1}` (assert).
- `H[L]   <= H[L-1] + 1e-9` (refinement monotonicity, Prop 4).
- `eps_star[L] <= eps_star[L-1] + 1e-9` (Prop 4).
- `lower[L] <= eps_star[L] <= upper[L]` for every $L$ (Thm 1).
- `bracket_width[L] <= W_STAR + 1e-9` for every $L$ (Cor 2).

### Runtime budget
- Twitch-EN, Cora, CiteSeer, PubMed: < 1 min each.
- ogbn-arxiv: ~10 min ($L$ rounds × hash of all edges per round).
- ogbn-proteins (112 tasks at one $L$): ~10 min.

### Critical reviewer rebuttal this provides
*"The bracket is not just a binary-classification curiosity but
quantitatively bounds the error of every admissible MPNN on a
real benchmark, with the upper bound achieved by the trivial
plug-in head on WL cells — no training."* — exactly what the GIN
and k-GNN papers stop short of providing.

## E6 — Cost comparison: bracket vs one training epoch

### Targets the reviewer concern
*"'Training-free' just hides where the cost goes; show that
evaluating the bracket is actually cheap compared to training."*
— answered by the $O(|V|)$ remark in §1, demonstrated here.

### Files to create
- `experiments/e6_cost.py` (note: `e6_nas.py` already exists for
  a different NAS-style experiment — name this `e6_cost.py` or
  fold both into one `e6_nas_cost.py`)
- `experiments/figures/e6_cost_ratio.pdf`
- `experiments/results/e6.json`

### Method

For each dataset $D$ in the menu:

1. Load $(X, y)$.
2. Build one $k$-means partition at $k = 16$ — call the cell
   vector $C$. (Other partition choices are interchangeable; the
   point is to compare *bracket evaluation* time, not
   partition-construction time.)
3. Time the four steps with `time.perf_counter`, median of 11
   runs, warm cache:
   - $T_{\mathrm{bracket}}$: one call to
     `bracket_from_cells(C, y)`.
   - $T_{\mathrm{CART}}$: one `DecisionTreeClassifier(max_leaf_nodes=16).fit(X, y)`.
   - $T_{\mathrm{kmeans}}$: one `KMeans(n_clusters=16).fit_predict(X)`.
   - $T_{\mathrm{LR}}$: one
     `LogisticRegression(max_iter=1).fit(OneHotEncoder().fit_transform(C.reshape(-1, 1)), y)`.
4. Report ratios
   $T_{\mathrm{bracket}} / T_{\mathrm{CART}}$,
   $T_{\mathrm{bracket}} / T_{\mathrm{kmeans}}$,
   $T_{\mathrm{bracket}} / T_{\mathrm{LR}}$.

### Output schema (`results/e6.json`)

```json
{
  "datasets": [{
    "name": "adult", "n": 45222, "d": 96, "k": 16,
    "t_bracket_ms":     0.42,
    "t_cart_ms":      180.0,
    "t_kmeans_ms":   4360.0,
    "t_lr_epoch_ms": 1230.0,
    "ratios": {"cart": 0.0023, "kmeans": 0.00010, "lr": 0.00034}
  }, ...]
}
```

### Figure
Grouped bar chart, log y-axis, four bars per dataset (bracket,
CART, KMeans, LR), with the $10^{-2}$ line drawn across.
Title: *"One bracket call ≪ one training epoch, across four
orders of magnitude in dataset size."*

### Verification gates
- `t_bracket_ms < t_cart_ms / 50` (i.e. bracket ≥ 50× cheaper
  than even the cheapest training step) on every dataset.
- `bracket(C, y).lower <= eps_star <= bracket(C, y).upper`
  (smoke check that the bracket we timed is still correct).

### Runtime budget
< 5 min total (the experiment *is* a timing benchmark, so it
must run in well-controlled conditions; use a noise-quiet
machine, disable Turbo Boost if possible).

### Relationship to the existing `e6_nas.py`
The current `e6_nas.py` aims at a NAS-style sweep across
architectures; this `e6_cost.py` (or a renamed sibling) is the
pure cost-comparison experiment. They are complementary; merge
them into one script if convenient.

## E7 — Real-data Proposition 7 concentration

### Targets the reviewer concern
*"Proposition 7 says the bracket concentrates to its population
analogue, but you only verified it on synthetic Monte Carlo.
Show it on a real dataset."* — empirical companion to
Proposition 7 + `verify_t4_population.py`.

### Files to create
- `experiments/e7_pop_concentration.py`
- `experiments/figures/e7_concentration.pdf`
- `experiments/results/e7.json`

### Method

1. Load UCI Adult, treat the full dataset as the "population"
   (so $\varepsilon^*_{\mathrm{full}}$, $H_{\mathrm{full}}$ are
   the ground-truth quantities).
2. Build a *fixed* partition $\Pi$ from a CART tree on the full
   data with `max_leaf_nodes = 16`. Freeze cell ids.
3. For each $n \in \{500, 1000, 5000, 10000, 20000, 45000\}$:
   1. Repeat $K = 200$ trials with seed `1000 + k`.
   2. Sub-sample $n$ rows uniformly without replacement.
   3. Compute $\varepsilon^*_{\mathrm{sub}}$, $H_{\mathrm{sub}}$
      on the same partition (restricted to the subsample).
   4. Record $\Delta_n^{(k)} = |\varepsilon^*_{\mathrm{full}} -
      \varepsilon^*_{\mathrm{sub}}| + |H_{\mathrm{full}} -
      H_{\mathrm{sub}}|$.
4. Compute analytical bound
   $\kappa(\delta, \eta) \sqrt{\log(4m/\delta_{\mathrm{conf}}) / n}$
   with $\delta = \min_C \mu(C)$, $\eta = \min_C \min(\bar P_C,
   1-\bar P_C)$, $m = 16$, $\delta_{\mathrm{conf}} = 0.05$.
5. Report empirical coverage:
   $K^{-1} |\{k : \Delta_n^{(k)} \le \mathrm{bound}\}|$.

### Output schema (`results/e7.json`)

```json
{
  "dataset": "adult", "m": 16, "delta_conf": 0.05,
  "delta_mass": 0.013, "eta_mean": 0.05,
  "full": {"eps_star": 0.1511, "H": 0.4823},
  "rows": [
    {"n": 500,   "K": 200, "delta_mean": 0.041, "delta_p95": 0.084, "bound": 0.51, "coverage": 1.0},
    {"n": 5000,  "K": 200, "delta_mean": 0.012, "delta_p95": 0.027, "bound": 0.16, "coverage": 1.0},
    {"n": 45000, "K": 200, "delta_mean": 0.000, "delta_p95": 0.001, "bound": 0.05, "coverage": 1.0}
  ]
}
```

### Figure
Single log–log plot: x-axis = $n$, y-axis = $\Delta_n^{(p95)}$
on the empirical side, plus the analytical bound as a dashed
line. Both should decay like $1/\sqrt n$.

### Verification gates
- `coverage >= 1 - delta_conf` for every $n$ (Prop 7 holds).
- `delta_mean[n]` strictly decreasing in $n$ (sanity).
- `delta_p95[n] <= bound[n]` (loose since Hoeffding is
  conservative).

### Runtime budget
< 1 min on UCI Adult.

### What this gives the reviewer that T4 does not
T4 (`verify_t4_population.py`) verifies Prop 7 on *constructed*
distributions with known $\mu, \pi$. E7 verifies it on a real
label whose true $\mu, \pi$ are estimated from the data itself,
showing that the assumptions $\mu(C) \ge \delta$ and $\bar P_C
\in [\eta, 1-\eta]$ are met on a benchmark practitioners
actually use.
