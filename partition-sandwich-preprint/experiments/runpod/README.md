# Two-phase RunPod split for the ogbn-arxiv slice of E3d-arch-full

This directory packages the operational plan agreed upon in the
adversarial review trilogy: **train on a GPU pod, dump the
penultimate embeddings `Z` to a persistent RunPod network volume,
stop the GPU pod, spin a cheap CPU pod against the same volume to
do the MiniBatchKMeans + bracket sweep, write the final JSON.**

This costs **~$0.15–0.30 total** instead of $0.50–1.10 if k-means
ran on the GPU pod's idle CPU, *and* leaves the `Z` artefacts
mounted as a reusable referee resource.

## Files

| File | Purpose | Where it runs |
|------|---------|---------------|
| [`e3d_arch_ogbn_train.py`](e3d_arch_ogbn_train.py) | Loop `train_one()` over 4 archs × 5 seeds; dump `Z_<arch>_<seed>.npz` + `meta.json` + `y_bin.npy` | GPU pod |
| [`e3d_arch_ogbn_kmeans.py`](e3d_arch_ogbn_kmeans.py) | Read the dumps; loop `kmeans_partition + bracket_from_cells` over `k_grid`; write `e3d_arch_full.ogbn_only.json` (exact schema match) | CPU pod (or local) |
| [`merge_to_5of5.py`](merge_to_5of5.py) | Merge with `experiments/results/e3d_arch_full.4of5.json` → `e3d_arch_full.5of5.json` | local |
| [`phase1_pod_bootstrap.sh`](phase1_pod_bootstrap.sh) | One-paste install + run for the GPU pod | GPU pod |
| [`phase2_pod_bootstrap.sh`](phase2_pod_bootstrap.sh) | One-paste install + run for the CPU pod | CPU pod |

## Wall-time + cost budget

| Phase | Hardware | Wall | Hourly | Cost |
|-------|----------|------|--------|------|
| 1. Train (4 archs × 5 seeds, full-batch transductive, 200 ep, hidden 128) | L4 24 GB (or RTX 3090) | 5–15 min | $0.20–0.40 | **~$0.05–0.10** |
| 2. K-means sweep (5 `k`-points × 20 cells, sklearn MiniBatchKMeans) | 8 vCPU CPU pod | 60–100 min | $0.05–0.10 | **~$0.05–0.15** |
| Network volume (~5 GB) | 10 GB allocation | hours-days | $0.07/GB/mo | **< $0.02** |
| **Total** | | **~75–115 min** | | **~$0.15–$0.30** |

## Provisioning checklist

1. **Create the network volume** (one-time):
   ```bash
   runpodctl volume create ogbn-arxiv-Z --size 10
   ```
   Or via the web UI under Storage → Network Volumes.

2. **Spin Phase-1 pod** (GPU):
   - GPU: `L4 24GB` or `RTX 3090`
   - vCPU: 4 (k-means doesn't run here)
   - Image: `runpod/pytorch:2.4.0-py3.11-cuda12.1.1-devel-ubuntu22.04`
   - Volume: mount `ogbn-arxiv-Z` at `/workspace/vol`
   - SSH or web-shell in, then:
     ```bash
     cd /workspace && git clone <your repo url> gnn_express
     bash gnn_express/partition-sandwich-preprint/experiments/runpod/phase1_pod_bootstrap.sh
     ```
   - When the log prints `[phase1] wrote meta.json + 20 Z_*.npz`, **STOP** the pod (do not delete). The volume persists.

3. **Spin Phase-2 pod** (CPU-only):
   - GPU: none
   - vCPU: 8, RAM 16 GB
   - Image: same Python image (CPU variant fine)
   - Volume: mount `ogbn-arxiv-Z` at `/workspace/vol` (same volume!)
   - SSH or web-shell in, then:
     ```bash
     cd /workspace && git clone <your repo url> gnn_express   # if pod is fresh
     bash gnn_express/partition-sandwich-preprint/experiments/runpod/phase2_pod_bootstrap.sh
     ```
   - When the log prints `[phase2] wrote e3d_arch_full.ogbn_only.json`, pull it locally:
     ```bash
     runpodctl send /workspace/vol/e3d_arch_full.ogbn_only.json   # on pod
     runpodctl receive <CODE>                                      # on laptop
     ```
   - **STOP** the pod.

4. **Locally**, merge to 5/5 and update the manuscript:
   ```bash
   cd partition-sandwich-preprint
   python experiments/runpod/merge_to_5of5.py \
       --four experiments/results/e3d_arch_full.4of5.json \
       --ogbn ~/Downloads/e3d_arch_full.ogbn_only.json \
       --out  experiments/results/e3d_arch_full.5of5.json
   # then apply the manuscript patch (see runpod/manuscript_patch_README.md
   # when it is generated by the 5/5 commit).
   make    # rebuilds main.pdf
   ```

5. **Optional**: keep `ogbn-arxiv-Z` for a week to make re-bracket
   experiments cheap (~$0.10 to re-run k-means with a new `k_grid` or
   a FAISS-based clusterer). Then `runpodctl volume delete ogbn-arxiv-Z`.

## Determinism notes

- Phase-1 dumps `Z` as `float16` by default to halve disk; the bracket
  is invariant under this rounding because it depends only on the
  discrete MiniBatchKMeans cluster IDs (the upcast to fp32 happens
  inside sklearn before any distance computation).
- `kmeans_partition` passes `random_state=seed` to MiniBatchKMeans, so
  Phase 2 is bit-deterministic per (`arch`, `seed`, `k`) given the
  same sklearn version on the same CPU vendor. We pin sklearn in the
  bootstrap to avoid silent ABI drift between phases.
- Train-time determinism is what it always was: fp32 on CUDA with
  default cuDNN, which is *not* bit-deterministic across reruns but
  is reproducible to ~3 decimals (matches the headroom of every other
  E3 table in the paper).

## Crash recovery

Per-dataset checkpointing in `e3d_arch_full.run_all()` does not apply
here because we only run one dataset (ogbn-arxiv). If the GPU pod dies
at cell *k*/20:
- the `.npz` for cells `0..k-1` are already on the volume,
- restart `phase1_pod_bootstrap.sh` but pass `--archs <remaining>` and
  `--seeds <remaining>` to skip what's already dumped (cf. the
  `--archs` / `--seeds` filter in `e3d_arch_ogbn_train.py`).

If the CPU pod dies mid k-means: just rerun `phase2_pod_bootstrap.sh`
from scratch (~60–100 min, idempotent).
